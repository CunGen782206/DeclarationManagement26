# API 模块设计（第一版）

基于系统流程，后端 API 拆分为：

1. Auth
   - 登录、获取当前用户信息。
2. Tasks
   - 申报任务管理（列表、新建、更新时间窗口）。
3. Declarations
   - 申报单创建、详情、我的申报分页。
4. Reviews
   - 预审/初审待办列表、执行审核动作（通过/不通过/驳回）。
5. Statistics
   - 统计筛选列表、Excel 导出、归档下载。
6. Users
   - 用户列表、新增、修改、删除、重置密码。

统一原则：
- Controller 仅负责入参与返回。
- 业务逻辑放 Service。
- 所有返回使用 ApiResponse<T>。
- 所有方法使用异步签名。
