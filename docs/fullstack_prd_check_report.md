# PRD 前后端完成度与可测性检查报告（全量）

> 检查对象：`src/DeclarationManagement.Api`、`frontend`
> 检查时间：2026-03-17
> 结论口径：以仓库内代码静态检查 + 本地可执行命令结果为准

## 1. 总体结论

- **是否“全部完成”**：**否**。主流程已具备，但仍存在影响 PRD 完整交付的缺口（前端功能缺页、权限边界、错误响应统一性、前后端字段对齐问题）。
- **是否“前后端可直接测试”**：**当前环境不可直接联调验证**。原因：
  - 后端无法构建（环境缺少 `dotnet`）。
  - 前端无法安装依赖（npm registry 403）。
- **是否“没有 bug”**：**不能成立**。已发现至少 1 个明确前后端字段映射 bug，会导致审核提交传错 `declarationId`。

## 2. 按 PRD 模块完成度检查

### 2.1 用户管理

- 后端提供用户列表/新增/修改/删除/重置密码接口，模块存在。`UsersController` 已实现完整 CRUD + reset。 
- 前端提供“用户管理”页面并路由到 `/users`。 
- **风险**：控制器仅 `[Authorize]`，未在控制器层增加超管策略限制（尽管前端路由做了 isSuperAdmin 拦截，后端仍缺强约束）。

### 2.2 申报任务管理

- 后端 `TasksController` 存在任务列表/创建/更新时间窗/启停。 
- **前端缺口**：路由中没有任务管理页面（仅 declarations/reviews/statistics/users），无法在 UI 完成“新建任务、改时间窗、启停”。

### 2.3 项目申报

- 后端具备创建、修改、提交、驳回后重提，且提交时校验任务时间窗。 
- 前端有申报列表 + 申报表单页面，字段覆盖 PRD 主体字段。 
- **缺口**：表单页未内置附件上传组件（PRD 要求附件上传下载）。

### 2.4 部门预审 / 初审

- 后端审核服务支持通过/不通过/驳回，并写审核记录与流程日志。 
- 前端有待审核列表与审核弹窗。
- **明确 bug**：前端审核页使用 `row.id` 作为 declarationId，但后端待审 DTO 字段是 `declarationId`（来自 `DeclarationId`），会导致传 0 或 undefined。 

### 2.5 导出与归档

- 后端支持统计查询、Excel 导出、归档 ZIP 导出。 
- 前端统计页提供查询、Excel 下载、归档下载按钮。
- **约束检查**：归档服务代码按“初审通过”过滤，符合 PRD 特别约束。

### 2.6 审核历史与流程日志

- 后端存在审核记录接口与流程日志接口。 
- **前端缺口**：当前页面未见“审核记录 + 流程日志”展示入口与组件。

## 3. 关键问题清单（阻塞“已完成且无 bug”）

1. **前后端字段映射 bug（P0）**
   - 审核页面提交动作使用 `row.id`，与后端返回 `declarationId` 不一致。

2. **任务管理前端缺失（P0）**
   - PRD 要求超管可配置任务时间窗；后端有接口，但前端无页面路由支撑。

3. **后端权限边界不足（P0）**
   - 用户/统计/任务控制器仅鉴权，不含超管策略，存在越权调用风险。

4. **统一错误响应未闭环（P1）**
   - 存在大量 `InvalidOperationException` 抛出，未见全局异常中间件统一包装失败响应。

5. **附件与历史日志前端体验未闭环（P1）**
   - 申报页缺附件上传下载入口；审核记录/流程日志无可视化页面。

## 4. 可直接测试性结论

当前仓库在本环境下**不能直接执行完整前后端联调测试**：

- 后端构建命令无法运行（系统缺 `dotnet`）。
- 前端依赖安装被 npm registry 403 拒绝，无法执行 `npm run build` / `npm run dev`。

因此，无法给出“没有 bug”的可执行验证结论；现阶段只能给出**静态审查结论：仍有功能缺口 + 已识别明确 bug**。

## 5. 建议下一步（按优先级）

1. 修复审核页 `declarationId` 取值（`row.declarationId`），并补上审核阶段显示字段对齐。
2. 增加超管任务管理前端页面（列表、新建、改时间窗、启停）。
3. 后端补充策略授权（至少用户、统计、任务 API 仅超管可调用）。
4. 增加全局异常处理中间件，统一失败响应结构。
5. 补齐附件上传/下载 UI 与审核历史/流程日志可视化。
6. 在具备 .NET SDK + 可用 npm registry 的环境重新执行联调回归。
