你正在开发一个“教学质量工程项目申报管理系统”。

## 技术栈
- 后端：.NET 10 Web API
- ORM：Entity Framework Core
- 数据库：SQL Server
- 前端：Vue 3 + TypeScript + Vite + Element Plus
- 鉴权：JWT
- 文件存储：本地磁盘
- Excel导出：ClosedXML
- PDF生成：QuestPDF

## 开发原则
1. 先设计，再编码。
2. 不要一次生成整个系统，必须按模块推进。
3. Controller 只负责接收请求和返回结果，不写复杂业务逻辑。
4. 业务逻辑写在 Application/Services。
5. Entity 不直接返回给前端，必须使用 DTO。
6. 所有数据库操作使用 EF Core 异步方法。
7. 所有接口返回统一响应结构。
8. 所有枚举使用明确命名，不要写魔法字符串。
9. 涉及审批流的状态变更，必须同时写入审核记录和流程日志。
10. 不确定需求时，不要自行猜测新增字段，先列出假设。
11. 每完成一步，先总结本步产物，再进入下一步。
12. 优先保证后端 API 和数据库稳定，再开始前端页面。

## 项目业务重点
- 用户管理
- 申报任务管理
- 项目申报
- 部门预审核
- 初审
- Excel导出
- PDF归档
- 附件上传下载
- 审核历史与流程日志

## 特别约束
- 预审权限和初审权限是多选，必须使用独立关联表，不可存逗号拼接字符串。
- 审核记录必须累加保存，不可覆盖。
- 驳回后允许申报人修改并重新提交。
- 统计导出按筛选结果导出。
- 归档只归档“初审通过”的表单。
